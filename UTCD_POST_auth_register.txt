UTCD Decision Table — POST /auth/register

Purpose
- This file is a text template you can mark with "O" (like your Excel matrix).
- It is derived from the current backend code:
  - AuthController.register(): returns 201 with { message, data }
  - RegisterDto: class-validator rules for request body
  - CaptchaGuard: optional depending on RECAPTCHA_ENABLED
  - AuthService.register(): conflict checks (email exists / legal consent)

How to use
1) Pick a column (UTCD01..UTCD12) and put "O" in the rows that apply.
2) For each UTCD column, mark exactly ONE expected outcome row (E..).
3) If you need more cases, copy UTCD12 to UTCD13, etc.

Legend
- Cxx = condition (input/environment)
- Exx = expected result

Conditions (rows)
C01  Request body is valid for all DTO validators
C02  Email format valid (IsEmail)
C03  Email is NOT disposable (IsNotDisposableEmail)
C04  Password valid (string, not empty, >=8, pattern)
C05  Full name valid (2-50 chars, letters/spaces)
C06  Phone valid (VN: ^0[3|5|7|8|9][0-9]{8}$)
C07  Role is registerable (CLIENT|BROKER|FREELANCER)
C08  acceptTerms is present (boolean)
C09  acceptPrivacy is present (boolean)
C10  acceptTerms == true
C11  acceptPrivacy == true
C12  Email does NOT already exist in DB
C13  RECAPTCHA_ENABLED == 'true'
C14  recaptchaToken provided (when captcha enabled)
C15  recaptchaToken is valid (captchaService.verifyRecaptcha == true)
C16  domainIds / skillIds (if provided) are arrays of strings

Expected outcomes (rows)
E01  201 Created; response has { message, data: AuthResponseDto }
E02  400 Bad Request; DTO validation failed
E03  400 Bad Request; captcha required or captcha invalid
E04  409 Conflict; email already used
E05  409 Conflict; must accept Terms + Privacy (legal consent)

Notes
- If captcha is disabled (RECAPTCHA_ENABLED != 'true'), CaptchaGuard always passes.
- The controller responds with 201 (Created) on success and includes a Vietnamese success message.
- Legal consent failure currently returns 409 Conflict (not 400) because AuthService uses ConflictException.
