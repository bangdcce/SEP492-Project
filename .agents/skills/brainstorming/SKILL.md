# brainstorming

Use this skill before creative work such as creating features, building components,
adding functionality, or changing behavior.

## Goal
- Clarify user intent.
- Confirm scope and constraints.
- Capture requirements and tradeoffs before implementation.

## Workflow
1. Restate the request and desired outcome.
2. List assumptions and constraints.
3. Identify risks and edge cases.
4. Produce a concise implementation direction.
5. Proceed to coding only after the above is clear.
