UC ID and Name: UC-03 Login
Created By: SonNT Date Created: 2026-01-22
Primary Actor: User (any registered role) Secondary Actors: Authentication System
Trigger: The user opens the Sign In screen and selects "Sign In".
Description: Allow a registered user to sign in with email and password and enter the correct dashboard for their role.
Preconditions: PRE1: The user already has an account.
PRE2: The user has verified their email address (if email verification is required).
PRE3: The user has an internet connection.
Post-conditions: POS1: The user is signed in and can access protected features.
POS2: The user is redirected to the correct dashboard based on their role.
POS3: The system records the sign-in event for security.
Normal Flow: A. Sign In Successfully
  1. The user opens the Sign In screen.
  2. The user enters their email and password.
  3. The user selects "Sign In".
  4. The system validates the credentials.
  5. If valid and the account is eligible to sign in, the system signs the user in.
  6. The system redirects the user to the correct dashboard.
Alternative Flows: A3. Missing or invalid input
  1. The user leaves email/password empty or enters an invalid email format.
  2. The system displays a message asking the user to correct the input.
A5. Email not verified
  1. The system informs the user that email verification is required.
  2. The user checks their email and completes verification, then tries again.
A6. Wrong email or password
  1. The system informs the user that the credentials are incorrect.
  2. The user retries or selects "Forgot password".
A7. Connection problem
  1. The system cannot complete the sign-in request due to a connection/server issue.
  2. The system shows an error message and the user can retry.
Exceptions: E1: The system is temporarily unavailable and sign-in cannot be completed.
Priority: High
Frequency of Use: High
Business Rules: BR1: The user must provide valid credentials.
BR2: If email verification is required, unverified users cannot sign in.
BR3: The system records sign-in activity for security.
Other Information: If the user forgets the password, they can use the "Forgot password" option.
Assumptions: N/A
