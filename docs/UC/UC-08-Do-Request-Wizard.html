UC ID and Name: UC-08 Do Request Wizard
Created By: SonNT Date Created: 2026-01-22
Primary Actor: Client Secondary Actors: Wizard System
Trigger: The client starts creating a new request using the guided wizard.
Description: Guide the client through a step-by-step form to describe their project needs and choose how to proceed (save draft, post, or invite).
Preconditions: PRE1: The client is signed in.
PRE2: The client has enough information about the project to answer the wizard questions.
PRE3: Some actions may require approved KYC (depending on policy).
Post-conditions: POS1: The client completes the wizard steps and the system prepares a request summary.
POS2: Depending on the chosen action, the request is saved as a draft or submitted for visibility.
Normal Flow: A. Complete Wizard and Continue
  1. The client opens the request wizard.
  2. The system shows step-by-step questions (type, industry, budget/timeline, features, title/description).
  3. The client answers each step and selects "Next".
  4. At the final step, the client chooses one option:
    • Save Draft
    • Post to Market
    • Find and Invite
  5. The system confirms the action and shows a success message.
Alternative Flows: A3. Missing required information
  1. The client skips a required step or leaves required fields empty.
  2. The system prevents moving forward until the information is provided.
A5. KYC is not approved (when required)
  1. The client tries to post or invite but KYC is not approved.
  2. The system informs the client to complete KYC or wait for approval.
  3. The client can still choose to save as draft.
Exceptions: E1: The system fails to save or submit due to a temporary error; the client can retry.
Priority: High
Frequency of Use: Medium
Business Rules: BR1: Only Clients can create requests using this wizard.
BR2: Required questions must be completed before submission.
BR3: Some actions may require approved KYC based on platform rules.
Other Information: A draft can be edited later before being posted.
Assumptions: N/A
