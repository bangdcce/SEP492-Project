UC ID and Name: UC-06 Manage Account
Created By: SonNT Date Created: 2026-01-22
Primary Actor: Signed-in User (Client/Broker/Freelancer/Admin) Secondary Actors: Account/Profile System
Trigger: The user opens the Account/Profile screen from the menu.
Description: Allow the user to view and update basic profile information such as name, phone number, bio, and avatar.
Preconditions: PRE1: The user is signed in.
PRE2: The user has an internet connection.
Post-conditions: POS1: The user can view current account/profile information.
POS2: If saved, the user's profile changes are stored and shown in the UI.
Normal Flow: A. View and Update Profile
  1. The user opens the profile screen.
  2. The system shows current profile information.
  3. The user selects "Edit".
  4. The user changes profile fields and optionally selects a new avatar image.
  5. The user selects "Save".
  6. The system validates and saves the updates.
  7. The system shows the updated profile information.
Alternative Flows: A5. Invalid avatar file
  1. The user selects a file that is not an image or is too large.
  2. The system shows an error and asks the user to choose another file.
A8. Cancel editing
  1. The user cancels editing.
  2. The system discards unsaved changes and shows the last saved profile.
Exceptions: E1: The system cannot save changes due to a temporary error; the user can retry.
Priority: High
Frequency of Use: Medium
Business Rules: BR1: Only signed-in users can access account management.
BR2: Profile changes must be validated before saving.
Other Information: Some profile fields may be read-only depending on role or policy.
Assumptions: N/A
