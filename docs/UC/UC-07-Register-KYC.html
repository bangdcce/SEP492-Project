UC ID and Name: UC-07 Register KYC
Created By: SonNT Date Created: 2026-01-22
Primary Actor: Signed-in User (Client/Broker/Freelancer) Secondary Actors: KYC Review Team (Admin/Staff)
Trigger: The user starts identity verification (KYC) from the application.
Description: Allow the user to submit identity information and documents for verification so the platform can approve sensitive actions.
Preconditions: PRE1: The user is signed in.
PRE2: The user has required documents and photos ready (ID front/back, selfie).
PRE3: The user has an internet connection.
Post-conditions: POS1: The KYC submission is recorded in the system.
POS2: The KYC status becomes "Pending" until reviewed.
POS3: The user can see the current KYC status in the application.
Normal Flow: A. Submit KYC Successfully
  1. The user opens the KYC screen.
  2. The user enters personal information (name, date of birth, document number, address).
  3. The user uploads required images (ID front, ID back, selfie).
  4. The user confirms and submits.
  5. The system validates the submission and stores it.
  6. The system shows a confirmation message that the review is pending.
Alternative Flows: A3. Missing required information
  1. The user does not provide required fields or documents.
  2. The system highlights missing items and blocks submission.
A5. Invalid image file
  1. The user uploads a file that is not an image or exceeds the size limit.
  2. The system shows an error and asks the user to upload a valid image.
A7. Resubmission after rejection
  1. The user's previous KYC was rejected.
  2. The user corrects the information and submits again.
Exceptions: E1: Upload or save fails due to a system error; the user can retry later.
Priority: High
Frequency of Use: Medium
Business Rules: BR1: Only signed-in users can submit KYC.
BR2: KYC status follows a review flow: Pending, Approved, or Rejected.
BR3: Uploaded documents must be clear images and may be size-limited.
Other Information: The review process may take time. The user can continue using non-restricted features while waiting.
Assumptions: N/A
