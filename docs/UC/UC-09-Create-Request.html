UC ID and Name: UC-09 Create Request
Created By: SonNT Date Created: 2026-01-22
Primary Actor: Client Secondary Actors: Request Management System
Trigger: The client finishes the wizard and confirms creation of a request.
Description: Create a new project request based on the information provided by the client.
Preconditions: PRE1: The client is signed in.
PRE2: The client has provided the required request information.
Post-conditions: POS1: A new request is created and saved in the system.
POS2: The client sees the request in their list of requests.
POS3: The system records the create action for tracking and security.
Normal Flow: A. Create Request Successfully
  1. The client completes the request wizard.
  2. The client chooses how to proceed (draft or visible request).
  3. The system creates the request and stores the details.
  4. The system shows a confirmation message.
  5. The client is redirected to the request list or request details screen.
Alternative Flows: A4. Save as draft
  1. The client chooses to save as a draft.
  2. The system saves the request without making it visible to others.
A6. Missing required fields
  1. The client attempts to create a request with missing required information.
  2. The system shows validation messages and asks the client to complete the fields.
Exceptions: E1: The system cannot create the request due to a temporary error; the client can retry.
Priority: High
Frequency of Use: Medium
Business Rules: BR1: Only Clients can create project requests.
BR2: Required information must be provided before creation.
BR3: The system tracks request creation activity.
Other Information: The client can edit a draft request later before posting it.
Assumptions: N/A
