UC ID and Name: UC-10 Invite Broker Directly
Created By: SonNT Date Created: 2026-01-22
Primary Actor: Client Secondary Actors: Broker (invited user)
Trigger: The client chooses a broker and selects "Invite".
Description: Allow a client to invite a broker to participate in a specific request.
Preconditions: PRE1: The client is signed in.
PRE2: The request exists and belongs to the client.
PRE3: The broker is available to be invited.
Post-conditions: POS1: The broker receives an invitation in their account/notifications.
POS2: The request shows that an invitation was sent.
POS3: The system records the invitation event.
Normal Flow: A. Invite Broker Successfully
  1. The client opens the request details.
  2. The client views suggested brokers (if available).
  3. The client selects a broker and chooses "Invite".
  4. The system sends the invitation and confirms success.
  5. The broker can later accept or respond to the invitation.
Alternative Flows: A2. Broker already invited
  1. The client tries to invite the same broker again.
  2. The system informs the client that the broker is already invited.
A4. No brokers available
  1. No brokers are available for invitation.
  2. The system shows an empty state and the client can try again later.
Exceptions: E1: The system cannot send the invitation due to a temporary error; the client can retry.
Priority: High
Frequency of Use: Medium
Business Rules: BR1: A client can only invite brokers for their own requests.
BR2: The same broker should not be invited multiple times for the same request.
BR3: Invitations and responses are tracked by the system.
Other Information: Inviting a broker does not automatically hire them; the broker must respond.
Assumptions: N/A
