UC ID and Name: UC-85 View System Log
Created By: An Date Created: 23/01/2026
Primary Actor: Admin Secondary Actors: System
Trigger: Admin navigates to "Audit Logs" or "System Activity".
Description: Allows the Admin to view an immutable history of critical system actions (e.g., Logins, Financial Transactions, Bans, Contract Signings). This is essential for security auditing, debugging, and dispute resolution.
Preconditions: PRE1: The user is logged in as Admin.
PRE2: The system has been recording logs.
Post-conditions: POS1: A list of log entries is displayed.
Normal Flow: A. View Logs
  1. Admin selects "Audit Logs" from dashboard.
  2. System retrieves log data from the database.
  3. System displays the logs with columns: Timestamp, Actor (User ID/Name), IP Address, Action Type (e.g., BAN_USER), Resource Affected, Details.
  4. Admin reviews entries.
Alternative Flows: A3. Filter Logs
  • At step 3, Admin filters by "Action: TRANSACTION" or "User: JohnDoe".
  • System updates the view.
A3. View Payload
  • At step 3, Admin clicks "View JSON" on a log entry to see the exact data change.
Exceptions: EXC-01: Database Timeout
  • Logs table is too large.
  • System suggests narrowing the date range.
Priority: Medium
Frequency of Use: Low (As needed)
Business Rules: BR-48 (History cannot be deleted).
BR-70 (Middleware recording).
