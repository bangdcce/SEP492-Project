UC ID and Name: UC-36 View Kanban Board
Created By: An Date Created: 23/01/2026
Primary Actor: Freelancer, Broker, Client, Staff Secondary Actors: System
Trigger: The user navigates to the "Tasks" or "Board" tab within a Project Workspace.
Description: Allows the user to view the project tasks organized in a Kanban board layout (columns: TODO, IN PROGRESS, DONE). This provides a visual overview of the project's progress and the status of individual work items.
Preconditions: PRE1: The user is logged into the system.
PRE2: The user is a member of the Project (or assigned Staff).
PRE3: The Project exists and is active.
Post-conditions: POS1: The Kanban board is displayed with all relevant tasks sorted into their respective columns.
POS2: Task details (assignee, priority, due date) are visible on the cards.
Normal Flow: A. View Kanban Board Successfully
  1. User selects the specific Project from their dashboard.
  2. User clicks on the "Task Board" tab.
  3. System retrieves the list of tasks associated with the project and their current statuses.
  4. System renders the Kanban board with columns (e.g., TODO, IN_PROGRESS, DONE).
  5. System populates the columns with task cards containing key information (Title, Assignee Avatar, Priority Badge).
  6. User views the board.
Alternative Flows: A3. Empty Board
  • If there are no tasks created for the project yet during retrieval (Step 3), the system displays an empty state message with a prompt to "Create First Task" (if the user has permission).
A6. Filter Tasks
  • At step 6, User applies a filter (e.g., "My Tasks", "High Priority").
  • System updates the board to show only tasks matching the criteria.
Exceptions: EXC-01: Data Retrieval Error
  • System fails to load tasks from the database.
  • System displays an error message and a "Retry" button.
Priority: High
Frequency of Use: High (Daily)
Business Rules: BR-33, BR-34 (Access control for moving items, though this UC is primarily View)
Other Information: Kanban view is the default view for Project Tasks.
Assumptions: N/A
