UC ID and Name: UC-84 Ban/Unban Account
Created By: An Date Created: 23/01/2026
Primary Actor: Admin Secondary Actors: System
Trigger: Admin decides to penalize a user or restore access.
Description: Allows the Admin to Ban a user (suspending their access to the system) or Unban a user (restoring access). Banning is usually a result of Dispute loss, fraud, or violations of Terms of Service.
Preconditions: PRE1: Admin is logged in.
PRE2: The target user account is active (for Ban) or banned (for Unban).
Post-conditions: POS1: The "isBanned" flag in the user table is updated.
POS2: The user is logged out (if online).
POS3: An email notification is sent to the user explaining the action.
Normal Flow: A. Ban User
  1. Admin views User Detail (UC-83).
  2. Admin clicks the "Ban Account" button.
  3. System displays a modal asking for confirmation and Reason (e.g., "Violated Policy").
  4. Admin enters reason and confirms.
  5. System updates status to Banned.
  6. System triggers BR-35 (Hide Public Requests).
B. Unban User
  1. Admin views a Banned User.
  2. Admin clicks "Unban Account".
  3. System asks validation "Are you sure?".
  4. Admin confirms.
  5. System restores access.
Alternative Flows: A4. Temporary Ban
  • At step 4, Admin selects a duration (e.g., 7 days) instead of permanent.
  • System schedules an automatic Unban job.
Exceptions: EXC-01: Cannot Ban Admin
  • Admin tries to ban another Super Admin.
  • System denies permission to prevent lockout.
Priority: Medium
Frequency of Use: Low
Business Rules: BR-35 (Banned user artifacts hidden, projects in review).
BR-36 (Banned users excluded from search).
Other Information: Ban actions are strictly logged in Audit Logs.
Assumptions: N/A
