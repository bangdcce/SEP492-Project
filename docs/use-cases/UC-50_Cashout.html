UC ID and Name: UC-50 Cashout
Created By: An Date Created: 23/01/2026
Primary Actor: Freelancer, Broker Secondary Actors: Admin, Bank/Payment System
Trigger: User clicks "Withdraw Money" in their Wallet.
Description: Allows the user to request a withdrawal of their available balance to an external bank account or payment method. The system deducts the amount from the internal wallet and creates a payout request for Admin processing or automated transfer.
Preconditions: PRE1: The user has a positive "Available Balance" (money not locked in Escrow).
PRE2: The user has a linked and verified withdrawal method (Bank/PayPal).
PRE3: The withdrawal amount exceeds the minimum threshold.
Post-conditions: POS1: Internal wallet balance is reduced by the withdrawal amount + fee.
POS2: A Payout Request is created with status "PENDING".
POS3: User receives a notification.
Normal Flow: A. Request Cashout
  1. User clicks "Withdraw" on Wallet page.
  2. System prompts for Amount and Destination Account.
  3. User enters amount and selects account.
  4. System calculates fees and estimates arrival time.
  5. User reviews and clicks "Confirm Withdraw".
  6. System validates balance sufficient.
  7. System creates the Payout Request record (UC-49).
  8. System locks the funds.
  9. System displays "Withdrawal Requested Successfully".
Alternative Flows: A6. Insufficient Funds
  • At step 6, User requests more than available balance.
  • System shows error: "Insufficient funds".
Exceptions: EXC-01: Account Suspended
  • User is banned or restricted.
  • System denies withdrawal and directs to support.
Priority: High
Frequency of Use: Low (Monthly)
Business Rules: BR-29 (Funds in Escrow cannot be withdrawn).
BR-31 (Split payouts logic applies to projects, but this UC is for the resulting balance).
Other Information: Withdrawals typically take 1-3 business days.
Assumptions: N/A
