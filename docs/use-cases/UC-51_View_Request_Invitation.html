UC ID and Name: UC-51 View Request Invitation
Created By: An Date Created: 23/01/2026
Primary Actor: Freelancer, Broker Secondary Actors: System
Trigger: User accesses their "Invitations" or "Job Offers" list.
Description: Allows the user (Broker/Freelancer) to view a list of invitations to collaborate on a Request or Project. These invitations are sent by Clients or Brokers. The user can view the basic details of the invitation and choose to Accept or Decline.
Preconditions: PRE1: The user is logged in.
PRE2: The user has received at least one invitation.
Post-conditions: POS1: A list of pending invitations is displayed.
Normal Flow: A. View Invitations
  1. User navigates to "Invitations" tab.
  2. System displays the list of pending invites.
  3. For each invite, the system shows: Project Name, Inviter Name, Date Sent, and status Badge (Pending).
  4. User can click to view full Request Details (UC-14/29).
Alternative Flows: A2. Accept Invite
  • At step 2, User clicks "Accept" on an invitation.
  • System adds the user to the candidate list or project team.
  • Invite status changes to "Accepted".
A2. Decline Invite
  • At step 2, User clicks "Decline".
  • System prompts for optional reason.
  • Invite status changes to "Declined".
Exceptions: EXC-01: Invite Expired
  • User tries to view/accept an old invite.
  • System notifies that the link has expired or the position is filled.
Priority: High
Frequency of Use: Medium
Business Rules: BR-40 (Client max invites), BR-36 (Banned users cannot be invited).
