UC ID and Name: UC-14 View Request Detail
Created By: SonNT Date Created:
Primary Actor: Client, Broker, Freelancer Secondary Actors:
Trigger: The user clicks on a specific Request title or the "View Details" button corresponding to a request item on the "My Requests" list (UC-13).
Description: Allows the user to view the comprehensive details of a specific Request. This includes the full set of specifications defined during the creation wizard (Product Type, Features, Tech Stack, Timeline), the financial details (Budget), and the current status (Draft, Private/Inviting, or Public/Open). This page acts as the central point for managing a specific request before it becomes a Project.
Preconditions: PRE1: The user is logged into the system.
PRE2: The user is currently viewing the list of their requests.
PRE3: The user Request exists and is valid.
Post-conditions: POS1: The Request Detail UI is fully rendered with all specifications and status indicators.
POS2: Context-aware action buttons are displayed based on the Request's status.
Normal Flow: A. View Request Details Successfully
  1. User clicks on a Request card or title in the "My Requests" list.
  2. System receives the Request ID and retrieves the detailed data from the database.
  3. System loads the Request Detail page.
  4. System displays the following sections:
  5. Header: Request Title, ID, Date Created, and Current Status tag.
  6. Specifications: The summary of answers provided in the Guided Input Wizard (Product Type, Scope, etc.).
  7. Financials: Budget range and payment preferences.
  8. Actions: Available buttons based on status.
  9. User reviews the information.
Alternative Flows: A3. Draft Request
  • 1. If the Request status is "Draft" it will only show to the client
Exceptions: EXC-01: Lost connection to the server or the Internet
  • The system displays an error message and prompts the user to retry action when connection is restored
Priority: High
Frequency of Use: High
Business Rules: BR-10, BR-11
