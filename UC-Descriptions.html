UC ID and Name: UC-45 Add Evidence
Created By: BangDC Date Created: 28/01/2026
Primary Actor: Client, Freelancer Secondary Actors: System
Trigger: User clicks "Upload".
Description: Uploading documents to the Evidence Vault.
Preconditions: PRE1: Dispute is open.
Post-conditions: POS1: The file binary is durably persisted in the Supabase Object Storage ('disputes' bucket) under a unique, collision-resistant path.
POS2: A corresponding metadata record (containing storage path, SHA-256 hash, MIME type, file size) is inserted into the PostgreSQL dispute_evidence table for management and retrieval.
Normal Flow: A. Secure Evidence Upload
  1. The User selects files from their local device context.
  2. The System calculates a cryptographic hash (SHA-256) of the file to ensure data integrity and detect duplicates.
  3. The System validates the file signature (MIME type and Magic Bytes) to prevent restricted file types.
  4. The System generates a unique, collision-resistant identifier for the file to ensure secure storage isolation.
  5. The System reserves a metadata record in the database.
  6. The System securely streams the file content to the Object Storage (Disputes Vault).
  7. The System performs an asynchronous malware scan via VirusTotal API.
  8. The System commits the transaction and triggers a notification event to relevant parties.
Alternative Flows: A.2 Duplicate File Detected
  1. System detects that a file with the exact same content hash already exists.
  2. System rejects the new upload and references the existing file to save storage.
A.3 Security Violation (Malware Detected)
  1. The VirusTotal scan returns a positive hit (>1 engine).
  2. The System immediately quarantines and purges the file from Storage.
  3. The System hard-deletes the database record.
  4. The User is flagged for security review.
  5. System sends a critical alert to Admins.
Exceptions: EXC-01: Lost connection to the server or the Internet
- The system displays an error message and prompts the user to retry action when connection is restored.

EXC-02: Storage Service Unavailable
- If the stream to the storage provider fails due to timeout or outages, the System rolls back the database record to ensure data consistency and alerts the user.
Priority: Medium
Frequency of Use: Low
Business Rules: BR-61
Other Information: Files are stored in a private bucket with RLS policies restricting access to participants only.
Assumptions: VirusTotal API quota is available.
