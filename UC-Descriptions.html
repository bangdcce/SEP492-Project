Trigger: A Project is marked as "Completed" or "Cancelled". System prompts parties to leave feedback.
Description: Allows users to rate their counterparts based on performance (Soft skills, Quality, Adherence to Schedule) and leave a text review. This data feeds into the Trust Score.
Preconditions: PRE1: Project status is "Completed".
PRE2: User has not already submitted a review for this project.
Post-conditions: POS1: Review is saved and visible on profile.
POS2: Trust Score is recalculated.
Normal Flow: A. Submit Review
  1. System displays the "Rate Your Experience" modal.

  3. User selects specific tags (e.g., "Good Communication", "High Quality").
  4. User enters a text comment (Optional).
  5. User clicks "Submit Review".
  6. System calculates the new average score for the target user.
  7. System updates the Trust Score (UC-80 logic).
  8. System publishes the review on the target user's profile.
Alternative Flows: A.3 Cancel Review
  1. User clicks the "X" or "Cancel" button on the modal.
  2. System closes the modal without saving data.
Exceptions: EXC-01: Lost connection to the server or the Internet
  • The system displays an error message and prompts the user to retry action when connection is restored.
EXC-02: Database Transaction Error
  • If the Trust Score calculation fails, the system logs the error and asks the user to try again later. Review is not saved.

Frequency of Use: Medium (Once per project completion)
Business Rules: BR-62
